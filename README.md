Each perk is its own ModSharp external module that compiles to a single `.dll`. A perk registers itself with a VIP core plugin through the shared `IVipShared` + `IVipPerkRegistry` interfaces, so you mix and match: drop in the perks you want, leave out the rest. There is nothing to configure in the core to add or remove a perk — its presence on disk is the toggle.

> Requires a VIP core plugin that publishes `IVipShared` and `IVipPerkRegistry` (`Vip.Core` / `Vip.Shared`). Without it each perk logs a warning at load and stays inactive.

## 🚀 Install

Copy the built perk modules you want into your ModSharp install (`<sharp>` = your `sharp` directory):

| From | To |
|------|----|
| `.build/modules/<Perk>/` | `<sharp>/modules/<Perk>/` |

Install only the perks you want — each is independent. Restart the server (or change map) to load. Per-perk JSON config is generated by the VIP core on first run; every perk ships disabled by default (`DefaultEnabled = false`) and must be enabled there.

## 🧩 Perks

Modules currently built from the solution:

| Module | Id | Description |
|--------|----|-------------|
| `Vip.Perk.AntiFlash` | `anti_flash` | Reduces flashbang blindness for VIP players. |
| `Vip.Perk.Armor` | `armor` | Grants kevlar on spawn. |
| `Vip.Perk.BunnyHop` | `bunny_hop` | Enables auto-bunny-hopping for VIP players. |
| `Vip.Perk.DefuseKit` | `defuse_kit` | Gives CT-side VIP players a free defuse kit on spawn. |
| `Vip.Perk.Health` | `health` | Grants extra HP on spawn. |
| `Vip.Perk.Healthshot` | `healthshot` | Grants a `weapon_healthshot` to VIPs each round. |
| `Vip.Perk.HitMarker` | `hitMarker` | Restricts floating damage numbers / hitmarker particles to VIPs only. |
| `Vip.Perk.KillScreen` | `kill_screen` | Flashes a color on-screen when the VIP player gets a kill. |
| `Vip.Perk.Money` | `money` | Gives VIP players a bonus cash amount on round start. |
| `Vip.Perk.PissmodGate` | `pissmodGate` | Restricts `!piss` to VIPs only. |
| `Vip.Perk.PoopGate` | `poopGate` | Restricts `!poop` to VIPs only. |
| `Vip.Perk.ReservedSlotsGate` | `reservedSlotsGate` | Tells ReservedSlots which players count as VIP. |
| `Vip.Perk.SmokeColor` | `smoke_color` | Changes the smoke grenade color for VIP players based on team. |
| `Vip.Perk.WeaponMenuGate` | `weaponMenuGate` | Restricts `!guns` / `!primary` / `!secondary` / `!<weapon>` to VIPs only. |

Additional perk sources are kept in the tree but excluded from the solution build (`Gravity`, `Items`, `Jumps`, `LifeSteal`, `NoFallDamage`, `Speed`, `Zeus`). Re-add them to `VipPerks.slnx` to build.

## 🔧 How it works

A perk implements `IVipPerk` (exposing its `Id`, `Description`, `DefaultEnabled`, and a list of `VipPerkSetting`s). On `OnAllModulesLoaded` the perk resolves `IVipShared` and `IVipPerkRegistry` from the module manager and registers itself; if either is missing it logs a warning and does nothing. At runtime the perk checks `IVipShared.IsVip(...)` and reads the player's per-perk `VipPerkPreferences` from the registry before applying its effect (e.g. setting armor on spawn). All VIP membership, preference storage, and config generation live in the VIP core — perks only contribute behaviour.

## 📦 Build

```bash
dotnet build -c Release
```

Each module builds to `.build/modules/<Perk>/<Perk>.dll`.

> The perk projects reference a local `Vip.Shared` project for the `IVipPerk` / `IVipShared` / `IVipPerkRegistry` contracts. Point the `ProjectReference` in each `*.csproj` at your copy of `Vip.Shared` before building.
